\section{Project Overview}
% =============================================================================

This project explores building, training, and analyzing deep neural networks for image recognition using the MNIST digit dataset. A convolutional neural network (CNN) is built from scratch in PyTorch, trained on 60,000 labeled digit images, and evaluated on a 10,000-image test set. The trained network is then examined by visualizing its learned filters and analyzing how they respond to input images. Transfer learning is applied to adapt the digit network to recognize three Greek letters using only 27 training examples. A Vision Transformer (ViT-style) network replaces the CNN as a drop-in replacement, demonstrating that transformer architectures can match CNN performance on MNIST. Two automated hyperparameter sweeps are conducted on Fashion MNIST: one varying transformer architecture dimensions and one comparing optimizers on the CNN. Three extensions are also implemented: analysis of pre-trained ImageNet network filters (ResNet18, VGG16, AlexNet), replacement of the first conv layer with hand-crafted Gabor filters, and a live webcam digit recognition application.

\medskip
\textbf{Extensions implemented:}
\begin{itemize}
    \item Extension 1: Pre-trained network conv layer analysis (ResNet18, VGG16, AlexNet)
    \item Extension 2: Gabor filter bank as the first convolutional layer
    \item Extension 3: Live webcam digit recognition application
\end{itemize}

\subsubsection{Task 1B -- CNN Architecture}

The network is a convolutional neural network derived from \texttt{torch.nn.Module} with the following layers in order: a Conv2d layer with 10 filters of size 5x5, a 2x2 max-pooling layer with ReLU, a second Conv2d layer with 20 filters of size 5x5, a Dropout layer with 50\% drop rate, a second 2x2 max-pooling layer with ReLU, a flattening operation followed by a fully connected Linear layer with 50 nodes and ReLU, and a final fully connected Linear layer with 10 nodes followed by log\_softmax.

\subsection{Task 3 -- Transfer Learning on Greek Letters}
% -----------------------------------------------------------------------------

The MNIST digit network is adapted for Greek letter recognition using transfer learning. The pre-trained weights are loaded, all parameters are frozen using \texttt{param.requires\_grad = False}, and the final layer \texttt{fc2} is replaced with a new \texttt{Linear(50, 3)} layer for the three classes: alpha, beta, gamma. Only the new layer is trained.

The Greek letter dataset contains 27 images (9 per class) at 133x133 pixels in color. A custom \texttt{GreekTransform} converts them to the MNIST format: RGB to grayscale, affine scale by 36/128, center crop to 28x28, and intensity inversion. Training uses batch size 5 and SGD with learning rate 0.01.

The network reaches near-perfect classification of the 27 training examples (training accuracy above 95\%) by approximately epoch 20 to 30. After 50 epochs the training loss is consistently below 0.05 and all 27 training examples are correctly classified. Training beyond 50 epochs yields no further improvement on the training set.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.65\textwidth]{projects/report_5/images/task3_training_curve.png}
    \caption{Task 3: Training loss per epoch for Greek letter transfer learning over 150 epochs. The loss drops sharply in the first 20--30 epochs as the new fc2 layer adapts to the three Greek classes, then plateaus near zero, indicating near-perfect classification of the 27 training examples.}
\end{figure}

The modified network structure is shown below. Only \texttt{fc2} changes from 10 to 3 output nodes.

\begin{verbatim}
DigitNetwork(
  (conv1): Conv2d(1, 10, kernel_size=(5, 5))
  (conv2): Conv2d(10, 20, kernel_size=(5, 5))
  (dropout): Dropout(p=0.5)
  (fc1): Linear(in_features=320, out_features=50)
  (fc2): Linear(in_features=50, out_features=3)   <- replaced
)
\end{verbatim}

Ten or more personal examples of each Greek letter were photographed, cropped to approximately 128x128, and run through the transfer-learned network. Images are named with a digit prefix matching the class index: 0 for alpha, 1 for beta, 2 for gamma.

\textbf{Custom Greek letter images:} \url{https://github.com/krhash/computer-vision/tree/main/5-deep-networks/data/greek_custom}

Increasing epochs beyond 150 shows diminishing returns, suggesting the bottleneck is the frozen feature representation rather than training duration.

\subsection{Task 4 -- Re-implement the Network Using Transformer Layers}
% -----------------------------------------------------------------------------

The CNN is replaced with a Vision Transformer (\texttt{NetTransformer}) that accepts the same input shape \texttt{(N, 1, 28, 28)} and produces the same output \texttt{(N, 10)}. The architecture has four components. First, a \texttt{PatchEmbedding} layer extracts overlapping 4x4 patches with stride 2 using \texttt{nn.Unfold}, producing 169 tokens, and projects each flattened patch into a 48-dimensional embedding. Second, learnable positional embeddings are added to the token sequence. Third, a stack of 4 \texttt{TransformerEncoderLayer} blocks with 8 attention heads and an MLP dimension of 128 processes the sequence. Fourth, tokens are averaged into a single vector, normalized, and passed through a two-layer MLP classifier.

Training uses AdamW with learning rate 0.001 and weight decay 1e-4 for 15 epochs. The default configuration is: patch\_size=4, stride=2, embed\_dim=48, depth=4, num\_heads=8.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.85\textwidth]{projects/report_5/images/task4_transformer_curves.png}
    \caption{Task 4: Transformer training and test loss (left) and accuracy (right) per epoch with default settings. The network reaches 97.37\% by epoch 4 and 98.94\% by epoch 15. Train and test accuracy remain close, indicating no overfitting.}
\end{figure}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.75\textwidth]{projects/report_5/images/task4_predictions.png}
    \caption{Task 4: First 9 MNIST test digits classified by the transformer network in a 3x3 grid. All 9 are correctly classified.}
\end{figure}

The transformer achieves 98.94\% test accuracy with default settings, comparable to the CNN at similar epoch counts.

\subsection{Task 5 -- Design Your Own Experiment}
% -----------------------------------------------------------------------------

Two separate hyperparameter sweeps are conducted on Fashion MNIST, which provides more room to observe performance differences than MNIST digits.

\subsubsection{Task 5A -- Transformer Hyperparameter Sweep}

55 configurations of \texttt{NetTransformer} are trained for 10 epochs each on Fashion MNIST, varying three dimensions: patch size \{2, 4, 7, 14\}, embedding dimension \{16, 32, 48, 96\}, and transformer depth \{1, 2, 4, 6\}. A round-robin linear search strategy is used: hold two dimensions at their default value and sweep the third, then rotate.

\textbf{Hypotheses before running:}
\begin{itemize}
    \item H1 (patch size): Medium patches (4 or 7) will outperform extremes. Patch=2 creates too many noisy tokens; patch=14 loses spatial detail.
    \item H2 (embed dim): Larger embeddings (48 or 96) will perform better. Fashion MNIST needs more representational capacity than digits.
    \item H3 (depth): Depth 2 to 4 will be the sweet spot. Depth=6 overfits and adds training time with diminishing returns.
\end{itemize}

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{projects/report_5/images/task5_experiment_results.png}
    \caption{Task 5A: Transformer sweep results on Fashion MNIST. Top-left: accuracy vs patch size. Top-right: accuracy vs embedding dimension. Bottom-left: accuracy vs transformer depth. Bottom-right: accuracy vs training time (all 55 runs). The orange dashed line marks baseline accuracy (89.49\%).}
\end{figure}

\begin{figure}[H]
    \centering
    \includegraphics[width=0.75\textwidth]{projects/report_5/images/task5_top_configs.png}
    \caption{Task 5A: Top 10 transformer configurations ranked by test accuracy on Fashion MNIST. The baseline (patch=4, embed=48, depth=4) achieved the highest raw accuracy at 89.49\%.}
\end{figure}

\textbf{Results and hypothesis discussion:}

H1 was partially supported. Patch=4 (baseline) achieved the highest accuracy at 89.49\%, and patch=2 was the worst performer (87.02--87.34\%) as predicted. However, patch=14 was more competitive than expected (88.26\%). H2 was not supported: embed=48 outperformed embed=96 consistently; larger embeddings added training time with no accuracy gain on this dataset size. H3 was supported: depth=2 performed nearly as well as depth=4 while reducing training time by 31\%. The best efficiency configuration is depth=2 with patch=4 and embed=48, achieving 88.96\% in 260 seconds compared to the baseline 89.49\% in 375 seconds.

\subsubsection{Task 5B -- CNN Optimizer Sweep}

26 configurations of \texttt{DigitNetwork} are trained on Fashion MNIST for 5 epochs each, varying three dimensions: optimizer \{SGD, Adam, AdamW, RMSprop\}, learning rate \{0.1, 0.01, 0.001, 0.0001\}, and SGD momentum \{0.0, 0.5, 0.9, 0.99\}.

\textbf{Hypotheses before running:}
\begin{itemize}
    \item H1 (optimizer): Adam and AdamW will converge faster and achieve higher accuracy than SGD.
    \item H2 (learning rate): Optimal LR differs per optimizer. SGD needs lr=0.01--0.1; Adam and AdamW work best at lr=0.001. Too high an LR will cause divergence.
    \item H3 (momentum): SGD with momentum=0.9 will outperform baseline momentum=0.5. Momentum=0.99 may oscillate.
\end{itemize}

\section{Reflection}
% =============================================================================

The most important lesson from this project was that training infrastructure matters as much as the architecture itself. Having a clean separation between the \texttt{Trainer}, \texttt{Evaluator}, and network classes made it straightforward to swap the CNN for the transformer and to reuse the same training loop for transfer learning and the experiment sweeps without rewriting any logic.

The transfer learning result on Greek letters was humbling. Only 9 training examples per class is genuinely not enough to generalize to handwritten samples with different stroke styles, even when the convolutional features from MNIST are reused. Beta's visual ambiguity demonstrated that the challenge is not the network depth or the learning rate but the insufficient training data diversity.

The hyperparameter sweeps produced a counterintuitive finding: the default transformer configuration was already near-optimal for Fashion MNIST, and increasing embedding dimension or depth provided no benefit. The CNN optimizer sweep was more productive: AdamW with lr=0.001 gave a clean +3\% improvement over the SGD baseline with the same training time.

The Gabor filter experiment was the most conceptually satisfying result. The 0.13\% accuracy gap between the Gabor network and the fully learned network is negligible, confirming that the first conv layer of a digit recognition network converges to something functionally equivalent to a classical oriented edge detector. Extension 1 visually confirmed this by showing that ResNet filters are almost indistinguishable from Gabor filters despite being trained end-to-end on ImageNet.
